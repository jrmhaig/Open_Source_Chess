The Open Source Chess Game is played via Github with pull requests. Anyone can propose or vote on a move for either black or white. The move with the highest score, calculated as up-votes minus down-votes, will be merged and all other pull requests for that turn will be closed. The exact schedule of merging moves is not fixed until I find out what works well.

Proposing moves

To propose a move:

  1. Clone this repository
  2. Edit the file CURRENT_GAME.pgn with your propsed move
  3. Create a pull request with the title in the format:
    • [White] 5. Bd4
    • [Black] 12. .. Qxa5
  4. Optionally add comments in the description
  5. Optionally(?) up-vote the pull request

Note: Please keep code pull requests separate from moves. Any pull requests for a move that change any file other than CURRENT_GAME.pgn will be closed automatically irrespective of how many votes it has so don't do it or you may cause the best move to be missed!

Voting on moves

To vote for a move simple give a 'thumbs up' to the pull request. Note that only votes on the pull request itself, underneath the description, will be counted and any votes on subsequent comments will be ignored. Analysis on moves as well as reactions to this analysis is encouraged but this will not be considered when selecting which move to merge. You may also give a 'thumbs down' to a move you consider to be bad. All other emoji responses will be ignored.
